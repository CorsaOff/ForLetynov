# Лабораторная работа: Удалённое управление Raspberry Pi по протоколам VNC, RDP и SSH

## 1. Подготовка к работе

Для успешного выполнения лабораторной работы вам понадобятся:
* **Общая сеть**: Raspberry Pi и ваш ПК должны находиться в одной локальной сети (или VPN).
* **Сетевые данные**: IP-адрес или Hostname вашей Raspberry Pi.
* **Учётные данные**: логин и пароль от учётной записи микрокомпьютера.

> 💡 *Если вы не знаете IP-адрес платы, определите его с помощью сканера сети (например, Advanced IP Scanner) или команды `hostname -I` в терминале платы.*

---

## 2. Настройка и подключение через VNC (TigerVNC)

1. **Скачайте клиент**: Перейдите в [репозиторий TigerVNC](https://github.com/TigerVNC/tigervnc/releases) и скачайте актуальную версию. 
   * Для **Windows**: скачайте файл `.exe`.
   * Для **Linux**: скачайте файл `.jar` и предварительно установите Java командой: `sudo apt install default-jre`.
2. **Запуск приложения**: 
   * В Windows запустите `.exe` файл двойным щелчком.
   * В Linux выполните в терминале: `java -jar VncViewer-<версия>.jar`.
3. **Настройка подключения**:
   * В поле **VNC-сервер** введите IP-адрес вашей платы (например, `192.168.129.150`).
   * Нажмите **Параметры** -> вкладка **Ввод** -> активируйте чекбокс **«Показывать точку при отсутствии курсора»** (чтобы указатель мыши не пропадал).
   * Нажмите **Соединить** (Connect). При появлении предупреждения безопасности нажмите **Да**.

   ![](vnc0.png)  
   ![](vnc1.png)

4. **Авторизация**: В окне аутентификации введите имя пользователя `user` и пароль `123`.

   ![аутентификация](vnc2.png)

5. **Фиксация результата**: Сделайте полноэкранный скриншот открывшегося рабочего стола Raspberry Pi. **Обязательно захватите системный трей вашего ПК с часами и датой.**

---

## 3. Альтернативное подключение через RDP

1. На локальном ПК под управлением Windows запустите стандартную утилиту **«Подключение к удаленному рабочему столу»** (mstsc).
2. Введите IP-адрес Raspberry Pi и учётные данные (`user` / `123`).
3. Сделайте скриншот успешного RDP-подключения (также с отображением системного трея вашего ПК).

---

## 4. Работа с файлами через SSH (Утилита SCP)

### Задание по передаче файлов (Upload):
1. Нажмите `Win + R`, введите `cmd` для открытия терминала на вашем ПК.
2. Подключитесь к плате по SSH и создайте в домашней директории папку, названную **Вашей фамилией** (например, `mkdir Ivanov`).
3. С помощью утилиты `scp` скопируйте файлы сделанных скриншотов (VNC и RDP) с вашего ПК в созданную на Raspberry Pi директорию.

### Задание по скачиванию файлов (Download):
1. Скопируйте файл с номером вашего варианта из общей директории `/letunov` на Raspberry Pi к себе на локальный компьютер.
2. Сохраните этот файл в папку с отчётом по лабораторной работе.
3. Сделайте скриншот терминала в момент выполнения этой операции (с системным треем).

---

## Справочник по командам SCP (Secure Copy Protocol)

Базовый синтаксис команды выглядит так:
```bash
scp [опции] [источник_файла] [куда_копировать]
```

### Примеры использования:

* **Загрузка файла с ПК на Raspberry Pi:**
  ```bash
  scp file.txt user@192.168.129.150:/home/user/Ivanov/
  ```
  *Примечание: Целевая папка должна быть создана заранее, так как `scp` не умеет создавать директории автоматически.*

* **Скачивание файла с Raspberry Pi на ПК (в текущую папку `./`):**
  ```bash
  scp user@192.168.129.150:/letunov/variant_1.txt ./
  ```

* **Скачивание файла в конкретную локальную папку:**
  ```bash
  scp user@192.168.129.150:/letunov/variant_1.txt C:/Users/Name/Documents/
  ```
